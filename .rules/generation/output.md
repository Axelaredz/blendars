# Output Contract

## Форматы ответа

| Тип | Формат |
|---|---|
| Новый .gd скрипт | Полный файл, с class_name, extends, type hints |
| Новый .tscn сцена | Полный текстовый формат Godot (text scene) |
| Новый .gdshader | Полный файл |
| Правка существующего | Diff: ±3 строки контекста, номера строк |
| Новый .tres ресурс | Полный текстовый формат |
| Lua (Nakama) | Полный файл |
| Docker/yml | Diff или полный — по размеру изменения |

## Ограничения
- Не больше 1 файла за ответ без явного запроса
- Исключение: связанная пара (.tscn + .gd) = допустимо как 1 ответ
- Файл > 200 строк → разбей на логические части, спроси порядок

## Naming
- Скрипты: `snake_case.gd` — `cyber_button.gd`, `player_controller.gd`
- Сцены: `snake_case.tscn` — `cyber_button.tscn`, `test_ui_kit.tscn`
- Ресурсы: `snake_case.tres` — `cyber_theme.tres`
- Шейдеры: `snake_case.gdshader` — `anime_outline.gdshader`

## Структура ответа
1. Путь к файлу (полный от корня проекта)
2. Код / содержимое
3. Краткое пояснение что сделано и почему (1-2 строки)
4. Если есть зависимости — указать какие файлы нужно создать/изменить ещё
